# Quickstart guide

Get the semantic search API running on your local machine in under 10 minutes.

## Prerequisites

Before starting, ensure you have:

- Python 3.13 or higher
- Docker and Docker Compose
- Git
- API keys for Google Gemini and Anthropic Claude

<Warning>
You need both **Google Gemini** and **Anthropic** API keys for the failover system to work properly. Get them from:
- [Google AI Studio](https://makersuite.google.com/app/apikey)
- [Anthropic Console](https://console.anthropic.com/)
</Warning>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the project and navigate to the source directory:

    ```bash
    git clone https://github.com/Pama205/sku-semantic-search.git
    cd sku-semantic-search
    ```
  </Step>

  <Step title="Create virtual environment">
    Set up a Python virtual environment and install dependencies:

    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: .\venv\Scripts\activate
    pip install -r requirements.txt
    ```

    <Info>
    The project uses key dependencies:
    - **fastapi** - High-performance async web framework
    - **google-generativeai** - Gemini embeddings and chat
    - **anthropic** - Claude fallback provider
    - **sqlalchemy** - Database ORM
    - **pgvector** - Python client for vector operations
    - **psycopg2-binary** - PostgreSQL adapter
    </Info>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root with your credentials:

    ```bash .env
    PROJECT_NAME="SKU Semantic Search"
    VERSION="1.0.0"
    
    # Database connection
    DATABASE_URL="postgresql://postgres:db_pass@localhost:5435/retail_rag"
    
    # AI provider credentials
    GEMINI_API_KEY="your_google_gemini_api_key_here"
    ANTHROPIC_API_KEY="your_anthropic_api_key_here"
    
    # JWT authentication
    JWT_SECRET="your_secure_random_secret_key_here"
    ALGORITHM="HS256"
    ACCESS_TOKEN_EXPIRE_MINUTES=30
    ```

    <Tip>
    Generate a secure JWT secret using: `python -c "import secrets; print(secrets.token_hex(32))"`
    </Tip>

    The configuration is loaded via Pydantic settings in `app/core/config.py`:

    ```python app/core/config.py
    from pydantic_settings import BaseSettings

    class Settings(BaseSettings):
        PROJECT_NAME: str = "SKU Semantic Search"
        VERSION: str = "1.0.0"
        
        DATABASE_URL: str
        GEMINI_API_KEY: str
        ANTHROPIC_API_KEY: str
        JWT_SECRET: str
        ALGORITHM: str = "HS256"
        ACCESS_TOKEN_EXPIRE_MINUTES: int = 30

        class Config:
            env_file = ".env"
            env_file_encoding = 'utf-8'
    ```
  </Step>

  <Step title="Start the database">
    Launch PostgreSQL with pgvector extension using Docker Compose:

    ```bash
    docker-compose up -d db
    ```

    This starts a PostgreSQL 16 container with pgvector:

    ```yaml docker-compose.yml
    db:
      image: pgvector/pgvector:pg16
      container_name: retail_rag_db
      ports:
        - "5435:5432"
      environment:
        - POSTGRES_USER=postgres
        - POSTGRES_PASSWORD=db_pass
        - POSTGRES_DB=retail_rag
      volumes:
        - postgres_data:/var/lib/postgresql/data
      healthcheck:
        test: ["CMD-SHELL", "pg_isready -U postgres"]
        interval: 10s
        timeout: 5s
        retries: 5
    ```

    <Info>
    The database runs on port **5435** to avoid conflicts with existing PostgreSQL installations.
    </Info>

    Wait for the health check to pass:

    ```bash
    docker-compose ps
    ```

    You should see the database status as "healthy".
  </Step>

  <Step title="Start the API server">
    Run the FastAPI application with hot reload:

    ```bash
    python -m uvicorn app.main:app --reload
    ```

    The server starts on `http://localhost:8000`. On startup, it automatically:
    - Creates the pgvector extension
    - Initializes database tables
    - Sets up CORS middleware

    ```python app/main.py
    from fastapi import FastAPI
    from sqlalchemy import text
    from app.core.config import settings
    from app.db.session import engine, Base

    def create_tables():
        with engine.connect() as conn:
            conn.execute(text("CREATE EXTENSION IF NOT EXISTS vector"))
            conn.commit()
        Base.metadata.create_all(bind=engine)

    app = FastAPI(title=settings.PROJECT_NAME, version=settings.VERSION)
    create_tables()
    ```
  </Step>

  <Step title="Seed the database (optional)">
    Load sample products to test semantic search:

    ```bash
    python seed_data.py
    ```

    This script:
    - Authenticates with the API (default credentials: `admin/admin123`)
    - Creates 40+ sample products across categories (beverages, dairy, cleaning, personal care, snacks, pet supplies)
    - Generates vector embeddings for each product

    <Warning>
    The seeding process calls the Gemini API for each product embedding. Add a delay between requests to avoid rate limits:
    
    ```python
    time.sleep(0.5)  # Wait 500ms between products
    ```
    </Warning>
  </Step>
</Steps>

## How the search works

The `/products/search` endpoint implements the RAG pattern in `app/api/endpoints/products.py:14`:

```python app/api/endpoints/products.py
@router.post("/search", response_model=SearchResultResponse)
def search_products(search_data: ProductSearchQuery, db: Session = Depends(get_db)):
    # 1. Vector similarity search in PostgreSQL
    products_db = ProductService.search_products(db, search_data.query, limit=search_data.limit)
    
    # 2. Assemble context from retrieved products
    context = ". ".join([f"{p.name}: {p.description}" for p in products_db])
    
    # 3. Generate AI recommendation using RAG
    ai_recommendation = LLMService.generate_answer(search_data.query, context)
    
    # 4. Return structured response
    return {
        "query": search_data.query,
        "recommendation": ai_recommendation,
        "results": products_db
    }
```

The LLM service handles multi-provider failover automatically (app/services/llm_service.py:64):

```python app/services/llm_service.py
@staticmethod
def generate_answer(query: str, context: str) -> str:
    prompt = (
        f"Eres un analista de Listo ERP. Basado en este contexto:\n{context}\n\n"
        f"Pregunta: {query}\nRespuesta profesional y breve:"
    )

    for entry in LLMService.LLM_CONFIG:
        provider = entry["provider"]
        for model_name in entry["models"]:
            try:
                if provider == "google":
                    res = LLMService._call_google(model_name, prompt)
                elif provider == "anthropic":
                    res = LLMService._call_anthropic(model_name, prompt)
                
                return f"[{provider.upper()} - {model_name}] {res}"
            
            except (APIStatusError, APIConnectionError) as e:
                print(f"⚠️ Error in {provider} ({model_name}): {e}")
                continue
    
    return "Lo sentimos, el servicio de recomendaciones no está disponible."
```

## Test the failover

To see the multi-LLM failover in action, temporarily set an invalid Gemini API key in your `.env` file. The system will automatically fall back to Claude:

```bash
GEMINI_API_KEY="invalid_key"
```

Restart the server and make a search request. The response will show:

```json
{
  "recommendation": "[ANTHROPIC - claude-3-haiku-20240307] ..."
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection errors">
    Ensure the database container is running and healthy:
    
    ```bash
    docker-compose ps
    docker-compose logs db
    ```
    
    Verify the `DATABASE_URL` in your `.env` matches the Docker Compose configuration.
  </Accordion>

  <Accordion title="API key errors">
    Check that your API keys are valid:
    - Gemini: https://makersuite.google.com/app/apikey
    - Anthropic: https://console.anthropic.com/settings/keys
    
    Ensure there are no extra spaces or quotes in the `.env` file.
  </Accordion>

  <Accordion title="Rate limit errors during seeding">
    Increase the delay between product insertions in `seed_data.py`:
    
    ```python
    time.sleep(1.0)  # Wait 1 second instead of 0.5
    ```
    
    Or use a paid API tier with higher rate limits.
  </Accordion>

  <Accordion title="Port already in use">
    If port 8000 or 5435 is already in use, change the ports in:
    - API: Add `--port 8001` to the uvicorn command
    - Database: Update the port mapping in `docker-compose.yml` and `DATABASE_URL`
  </Accordion>
</AccordionGroup>
