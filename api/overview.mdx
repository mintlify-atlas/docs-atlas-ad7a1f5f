## Introduction

The SKU Semantic Search API is a powerful REST API that enables intelligent product search using semantic embeddings and AI-powered recommendations. The API leverages vector similarity search and large language models to provide contextual product recommendations.

## Base URL

All API endpoints are relative to the base URL:

```
http://localhost:8000
```

## Authentication

The SKU Semantic Search API uses **OAuth2 with Bearer tokens** for authentication. Currently, authentication is implemented for the `/auth/login` endpoint, which returns a JWT token. However, product endpoints (`/products/` and `/products/search`) do not require authentication in this proof-of-concept implementation.

### Example login request

```bash
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=admin123"
```

## Common headers

Include these headers in your API requests:

| Header | Value | Required | Description |
|--------|-------|----------|-------------|
| `Content-Type` | `application/json` | Yes | Specifies JSON request body |
| `Accept` | `application/json` | No | Specifies JSON response format |

## Response format

All API responses are returned in JSON format with appropriate HTTP status codes.

### Success response

```json
{
  "id": 1,
  "name": "Example Product",
  "description": "Product description",
  "category": "Electronics"
}
```

### Error response

```json
{
  "detail": "Error message describing what went wrong"
}
```

## Error codes

The API uses standard HTTP status codes to indicate success or failure:

| Status Code | Meaning | Description |
|-------------|---------|-------------|
| `200` | OK | Request successful |
| `201` | Created | Resource created successfully |
| `400` | Bad Request | Invalid request parameters |
| `401` | Unauthorized | Authentication required or failed |
| `404` | Not Found | Resource not found |
| `422` | Unprocessable Entity | Validation error in request body |
| `500` | Internal Server Error | Server-side error |

## Rate limiting

Currently, the API does not enforce rate limiting. This may change in future versions.

## CORS

The API supports Cross-Origin Resource Sharing (CORS) and accepts requests from:

- `http://localhost:4200` (default Angular development server)

All HTTP methods and headers are allowed for configured origins.

## Health check

You can verify the API is running by accessing the root endpoint:

```bash
curl http://localhost:8000/
```

**Response:**
```json
{
  "message": "API de Búsqueda Semántica de SKUs lista y Base de Datos conectada"
}
```

## API endpoints

The API provides the following endpoints:

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Health check and API status |
| `POST` | `/auth/login` | Authenticate and obtain JWT token |
| `POST` | `/products/` | Create a new product with embeddings |
| `POST` | `/products/search` | Semantic search with AI recommendations |
