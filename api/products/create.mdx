## Endpoint

```
POST /products/
```

Creates a new product in the database and automatically generates semantic embeddings for intelligent search. The product's name and description are combined to create a vector embedding that enables semantic similarity search.

<Note>
This endpoint requires authentication. Include a valid Bearer token in the Authorization header.
</Note>

## Request body

<ParamField body="name" type="string" required>
  The product name or SKU identifier
</ParamField>

<ParamField body="description" type="string" required>
  Detailed description of the product, its features, and use cases
</ParamField>

<ParamField body="category" type="string" required>
  Product category for classification (e.g., "Electronics", "Clothing", "Home & Garden")
</ParamField>

import requests

## Automatic embedding generation

When you create a product, the system automatically:

1. **Combines text:** Concatenates the product name and description
2. **Generates embedding:** Creates a high-dimensional vector representation using a sentence transformer model
3. **Stores vector:** Saves the embedding in a PostgreSQL database with pgvector extension
4. **Enables search:** Makes the product immediately searchable via semantic similarity

This process happens transparently in the background. You don't need to manually generate or provide embeddings.

## Best practices

<Tip>
**Write detailed descriptions:** The quality of semantic search depends on the richness of product descriptions. Include features, use cases, and relevant keywords to improve search accuracy.
</Tip>

<Warning>
**Avoid duplicate products:** Creating duplicate products will result in redundant search results. Check for existing products before creating new ones.
</Warning>
