This guide walks you through setting up the SKU Semantic Search application from scratch.

## Prerequisites

Before you begin, ensure you have the following installed on your system:

<Steps>
  <Step title="Install Docker and Docker Compose">
    Docker is required to run the PostgreSQL database with pgvector extension.

    - **Docker**: Download from [docker.com](https://www.docker.com/get-started)
    - **Docker Compose**: Included with Docker Desktop, or install separately

    Verify installation:
    ```bash
    docker --version
    docker-compose --version
    ```
  </Step>

  <Step title="Install Python 3.11">
    The application requires Python 3.11 or later.

    ```bash
    python --version  # Should show 3.11 or higher
    ```

    <Note>
      If you need to manage multiple Python versions, consider using [pyenv](https://github.com/pyenv/pyenv)
    </Note>
  </Step>

  <Step title="Obtain API keys">
    You'll need API keys for the embedding and generation services:

    - **Gemini API Key**: Get from [Google AI Studio](https://makersuite.google.com/app/apikey)
    - **Anthropic API Key**: Get from [Anthropic Console](https://console.anthropic.com/)

    <Warning>
      Keep your API keys secure and never commit them to version control
    </Warning>
  </Step>
</Steps>

## Clone the repository

Clone the SKU Semantic Search repository to your local machine:

```bash
git clone <repository-url>
cd sku-semantic-search
```

## Database setup with Docker

The application uses PostgreSQL with the pgvector extension for storing and querying vector embeddings.

<Steps>
  <Step title="Review docker-compose.yml">
    The `docker-compose.yml` file defines two services:

    ```yaml docker-compose.yml
    services:
      api:
        build: .
        container_name: retail_rag_api
        ports:
          - "8000:8000"
        environment:
          - DATABASE_URL=postgresql://postgres:db_pass@db:5435/retail_rag
          - GEMINI_API_KEY=${GEMINI_API_KEY}
          - JWT_SECRET=una_clave_secreta_muy_segura_para_el_mvp
        depends_on:
          db:
            condition: service_healthy
        command: uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

      db:
        image: pgvector/pgvector:pg16
        container_name: retail_rag_db
        ports:
          - "5435:5432"
        environment:
          - POSTGRES_USER=postgres
          - POSTGRES_PASSWORD=db_pass
          - POSTGRES_DB=retail_rag
        volumes:
          - postgres_data:/var/lib/postgresql/data
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U postgres"]
          interval: 10s
          timeout: 5s
          retries: 5
    ```

    Key points:
    - Database runs on port **5435** (mapped from container's 5432)
    - Uses the official **pgvector/pgvector:pg16** image
    - Health checks ensure database is ready before API starts
  </Step>

  <Step title="Create environment file">
    Create a `.env` file in the project root with your API keys:

    ```bash
    touch .env
    ```

    Add the following content:

    ```bash .env
    DATABASE_URL=postgresql://postgres:db_pass@localhost:5435/retail_rag
    GEMINI_API_KEY=your_gemini_api_key_here
    ANTHROPIC_API_KEY=your_anthropic_api_key_here
    JWT_SECRET=una_clave_secreta_muy_segura_para_el_mvp
    ```

    <Info>
      The `DATABASE_URL` uses `localhost:5435` for local development when running the API outside Docker
    </Info>
  </Step>

  <Step title="Start the database">
    Launch the PostgreSQL container:

    ```bash
    docker-compose up db -d
    ```

    Verify the database is running:

    ```bash
    docker-compose ps
    ```

    You should see the `retail_rag_db` container with status "healthy".
  </Step>
</Steps>

## Python environment setup

Set up a virtual environment and install dependencies.

<Steps>
  <Step title="Create virtual environment">
    ```bash
    python -m venv venv
    ```

    Activate the virtual environment:

    <CodeGroup>
      ```bash macOS/Linux
      source venv/bin/activate
      ```

      ```bash Windows
      venv\Scripts\activate
      ```
    </CodeGroup>
  </Step>

  <Step title="Install dependencies">
    The project uses a `requirements.txt` file with all necessary packages:

    ```bash
    pip install -r requirements.txt
    ```

    Key dependencies include:
    - **fastapi**: Web framework
    - **uvicorn**: ASGI server
    - **sqlalchemy**: ORM for database operations
    - **pgvector**: Vector similarity search
    - **google-generativeai**: Gemini API client
    - **anthropic**: Claude API client
    - **python-jose**: JWT token handling
    - **passlib**: Password hashing
  </Step>

  <Step title="Initialize database tables">
    The application automatically creates tables and enables the pgvector extension on startup:

    ```python app/main.py
    def create_tables():
        with engine.connect() as conn:
            conn.execute(text("CREATE EXTENSION IF NOT EXISTS vector"))
            conn.commit()
        Base.metadata.create_all(bind=engine)
    ```

    This function runs when the FastAPI application starts.
  </Step>
</Steps>

## Run the application

Start the development server:

```bash
uvicorn app.main:app --reload
```

The API will be available at:
- **API**: http://localhost:8000
- **Interactive docs**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc

<Info>
  The `--reload` flag enables auto-reload on code changes during development
</Info>

## Using Docker Compose (alternative)

You can also run both the API and database using Docker Compose:

```bash
docker-compose up
```

This will:
1. Build the API container from the Dockerfile
2. Start the PostgreSQL database
3. Wait for the database to be healthy
4. Start the API server on port 8000

<Note>
  When using Docker Compose, make sure your `.env` file is in the project root and contains the `GEMINI_API_KEY`
</Note>

## Verify installation

Test that the API is running correctly:

```bash
curl http://localhost:8000/
```

You should receive:

```json
{
  "message": "API de Búsqueda Semántica de SKUs lista y Base de Datos conectada"
}
```
